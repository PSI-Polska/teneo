Step 1: Environment Setup

The first step in the tutorial is to setup the environment
  • Add runtime plugin to dependencies of plugin.xml
  • Create Empty Database
  • Add Logging

Add runtime plugin to dependencies of plugin.xml

To enable the runtime layer the org.elver.store.hibernate.runtime plugin needs to be added to the plugin dependencies of the plugin.xml (see image below).

In addition the following Eclipse-RegisterBuddy: org.elver.store.hibernate.runtime, needs to be added to the MANIFEST.MF of the Library model project (see here why).

In addition your database JDBC driver should be added to classpath of the plugin and possible the packages of the jdbc driver have to be made visible to other plugins.

Create Empty Database

The runtime layer will automatically create tables and foreign constraints. However it will not automatically create a database. For this tutorial you need to manually create an empty database in your database server, the database should have the name: library.

Add Logging

To view the log messages of Hibernate and Elver you have to set the log4j properties. A simple sample log4j.properties file can be downloaded here. The log4j.properties file has to be placed directly in the src folder of the library model project.
