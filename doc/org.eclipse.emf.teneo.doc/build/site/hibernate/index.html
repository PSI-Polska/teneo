Elver Persistency moves to Eclipse.org as Teneo within EMFT

Elver persistency is in the process of moving to eclipse.org. Elver persistency will be continued as the Teneo project in the EMFT project. The current status:

  • The source code has been moved to Eclipse cvs.
  • Currently we are working on the build process.
  • Support is given on both the EMFT newsgroup as the Elver mailinglist. When the build process at eclipse.org is operational, support will move completely to the EMFT newsgroup.

EMF Persistency Solution using Hibernate 3.1

The EMF - Hibernate integration currently supports a number of important EMF features and standard Hibernate features. EMF objects can be persisted in most major relational databases and retrieved using an EMF resource approach or HQL queries.

Current version: 0.7.2 ALPHA (last updated 7 June 2006) for Eclipse 3.2*/EMF 2.2*/Hibernate3.2 (and Hibernate 3.1.3) and jdk 1.4 and later, Status, Features, Changelog

Releases are not guaranteed to be backward compatible. When upgrading: please read the Changelog and the Upgrading pages.

For troubleshooting please visit the Known Issues and Troubleshooting pages.

*This release should also work for Eclipse 3.1/EMF 2.1, however only limited testing has been done for these version.

Hibernate

The EMF - Hibernate integration is a layer over the Hibernate product. The used Hibernate version is 3.1.3. For more information visit Hibernate at http://www.hibernate.org.

EMF

The Eclipse Modeling Framwork (EMF) is a modeling and code generation framework which generates source code from different sources: an UML diagram or a XML Schema. For more information see here.
