Running the GMF Editor

  • Link to the database

  • Possible issues

This section replaces the 'Running the diagram' part of the GMF tutorial.

Link to the database

Teneo requires connection information as well as which epackages are to be persisted in the database. For this tutorial this information is set in a property file.

You can download a property file with most information set for the mindmap tutorial here.

Remarks:

  • The database properties in the property file have to be changed to contain your own database connection information.
  • The property file is set to use the database 'mindmap'. This database has to exist (but can be empty).

Running the diagram

To run the diagram start a second workbench and create a project. Then save the property file in the project. The property file must have the extension mindmap.

Next right click on the property file and select 'Initialize mindmap_diagram diagram file'.

Just select the default choices in the wizard. After pressing finish the tables in the database are created and the mindmap diagram editor is opened.

Now you can create topics and relate them. When you press save the data is stored in the database.

You can close the editor and re-open it by right-clicking on the mindmap_diagram file. The open logic assumes that the mindmap_diagram file and the .mindmap file are in the same location.
