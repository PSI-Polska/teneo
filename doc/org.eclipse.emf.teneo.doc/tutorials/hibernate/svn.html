Subversion

  • Projects in Subversion

The source code of the EMF - Hibernate integration is stored in a Subversion repository. The subversion url is: svn://svn.elver.org/elver with anonymous login.

The source code in subversion is updated on a daily basis and contains experimental new features which can (temporarily) break already existing functionality and tests. There is no guarantee that the code in subversion will work in anyway.

Projects in Subversion

The relevant Hibernate related projects within the repository are:

  • org.elver.store: this project contains general code used by the JPOX as well as the Hibernate runtime layer.
  • org.elver.store.eclipse: this project contains general code for the eclipse menu options used by the JPOX as well as the Hibernate runtime layer.
  • org.elver.store.annotations: this project contains code for reading and generating annotations.
  • org.elver.store.hibernate: this project contains the EMF - Hibernate mapping and runtime layer.
  • org.elver.store.hibernate.eclipse: this project contains the resource utility Eclipse plugins.
  • org.elver.store.samples: this project contains the examples and model information used in the testcases.
  • org.elver.store.commontest: this project contains the test case part which is common for Hibernate and JPOX
  • org.elver.store.hibernate.test: the hibernate specific part of the tests
  • org.elver.store.tutorial: the tutorials used on this website

Other projects in subversion are related to an integration between EMF and JSF and Hibernate.

The projects have dependencies which are managed by the plugin dependencies in the Manifest.MF

Sometimes projects will have a dependency to a org.hibernate plugin project (not present in svn). This is used to debug hibernate. When checking out the development projects these dependencies can be replaced by the corresponding hibernate jar file.

Within each project there is a sub folder called trunk which contains the latest version. Other versions are present in the branch/version subfolders.

We use the standard Eclipse code format with the line length set to 120, the formatter profile can be downloaded here.
